# 邮件服务配置说明

## 📧 概述

当前找回密码功能已支持真实邮件发送。默认情况下，邮件服务处于**测试模式**（不会真实发送邮件，但会在API响应中返回验证码供测试）。

## 🚀 启用真实邮件发送

### 步骤1: 复制配置文件

```bash
cp .env.example .env
```

### 步骤2: 编辑配置文件

打开 `.env` 文件，配置您的邮件信息：

```env
# 启用真实邮件发送
ENABLE_EMAIL_SENDING=true

# SMTP服务器配置（以QQ邮箱为例）
SMTP_SERVER=smtp.qq.com
SMTP_PORT=587
SMTP_USERNAME=your-email@qq.com
SMTP_PASSWORD=your-authorization-code
SENDER_EMAIL=your-email@qq.com
```

### 步骤3: 获取邮箱授权码

#### QQ邮箱
1. 登录QQ邮箱网页版
2. 点击「设置」→「账户」
3. 找到「POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务」
4. 开启「IMAP/SMTP服务」
5. 按提示发送短信验证
6. 复制生成的**授权码**（不是登录密码！）

#### Gmail邮箱
1. 登录Google账号
2. 进入「安全性」设置
3. 开启「两步验证」
4. 创建「应用专用密码」
5. 使用生成的专用密码作为 `SMTP_PASSWORD`

#### 163邮箱
1. 登录163邮箱网页版
2. 点击「设置」→「POP3/SMTP/IMAP」
3. 开启「IMAP/SMTP服务」
4. 获取授权码

### 步骤4: 重启服务

```bash
# 停止当前服务（Ctrl+C）
# 重新启动
python run.py
```

## 📋 各邮箱SMTP配置参考

| 邮箱服务商 | SMTP服务器 | 端口 | 说明 |
|----------|-----------|------|------|
| QQ邮箱 | smtp.qq.com | 587 | 需要授权码 |
| Gmail | smtp.gmail.com | 587 | 需要应用专用密码 |
| 163邮箱 | smtp.163.com | 465/994 | 需要授权码 |
| 126邮箱 | smtp.126.com | 465 | 需要授权码 |
| Outlook | smtp.office365.com | 587 | 直接使用登录密码 |
| Yahoo | smtp.mail.yahoo.com | 587 | 需要应用专用密码 |

## 🧪 测试模式（默认）

如果您只想测试功能而不配置真实邮箱：
- 保持 `ENABLE_EMAIL_SENDING=false`
- 请求验证码时，API会直接返回验证码
- 在前端输入框中输入返回的验证码即可继续

## 📝 邮件内容

发送的邮件包含：
- 清晰的HTML格式
- 6位验证码大字体显示
- 15分钟有效期提示
- 精美的UI设计

## 🔧 故障排除

### 问题1: 认证失败
**原因**：授权码错误或未开启SMTP服务
**解决**：重新获取授权码，确保SMTP服务已开启

### 问题2: 连接超时
**原因**：防火墙或网络限制
**解决**：检查网络连接，尝试更换端口

### 问题3: 邮件被识别为垃圾邮件
**原因**：新邮箱发送邮件
**解决**：让用户在邮箱中标记为"非垃圾邮件"

## 📊 数据记录

所有邮件发送记录都会永久保存到 `password_reset_emails` 表，包含：
- 收件人邮箱
- 验证码
- 发送时间
- 过期时间
- 是否使用
- 关联用户ID

## 💡 安全建议

1. **不要将 `.env` 文件提交到代码仓库**
2. 定期更换邮箱授权码
3. 监控邮件发送日志
4. 考虑使用专业邮件服务（SendGrid、Mailgun等）
